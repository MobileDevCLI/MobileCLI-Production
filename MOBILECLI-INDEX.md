## DITTO ARCHITECTURE (AI UI Control)

**What is DITTO?**
DITTO allows the AI to control the app's UI at runtime via terminal commands.

**Key Components:**
- `DittoJsBridge` - JavaScript bridge for terminal commands
- WebView overlay - Morphable UI layer
- Profile system - Shareable UI configurations

**Full Documentation:** `~/MobileCLI-Baseline/docs/intellectual-property.md` (Section 2)

**Commands:**
```bash
mobilecli-ui set-background "#1a1a2e"
mobilecli-ui set-text-size 32
mobilecli-ui save-profile my-setup
mobilecli-ui load-profile my-setup
mobilecli-ui browser   # Profile browser UI
```

**Repo:** `https://github.com/MobileDevCLI/ditto-store`

## MULTI-AGENT SYSTEM

**What is it?**
Multiple Claude Code instances communicating via `~/.claude/` JSONL files.

**Full Documentation:** `~/MobileCLI-Lab/MULTI-AGENT.md`

**Commands:**
```bash
agent discover   # Find all Claude sessions
agent read <id>  # Read another agent's conversation
agent send <id>  # Send message to another agent
agent hub        # Supervisor mode
```
